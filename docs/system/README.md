# System Directory

This folder contains configuration and integration documentation for the AI agents that operate this vault and Paul's automation stack. Files here are maintained by Claude Code and Hermes — not edited manually unless there is a structural change.

---

## Files in This Directory

### `claude-handoff.md`
A snapshot of the full system state: vault structure, GitHub repo, Telegram config, Hermes skills list (93 skills), all MCP server configs, key file paths, and Hermes `.env` variable names. Auto-generated by Hermes. Source of truth for system topology.

### `Claude-Integration.md`
A comprehensive guide for Hermes explaining how Claude Code is configured. Covers all active MCP server connections, Claude's delegation model (bash aliases and which models handle what tasks), delegation rules, how Hermes should communicate with Claude, and the full list of Claude skills and commands relevant to Manyhat Automations.

### `Daily-Sync.md`
The daily synchronisation protocol between Claude Code and Hermes. Defines the exact markdown templates each agent writes to the vault, where files are stored, how file rotation works at midnight, sync frequency, GitHub push procedures, and the full cron agent schedule (3am–9pm).

### `Claude-Transfer/`
Directory for session transfer files — used when Claude needs to hand off context between sessions or to a fresh instance. Contents are ephemeral and may be empty.

---

## How This System Works

```
Paul ←→ Hermes (Telegram) ←→ Vault files ←→ Claude Code
```

- **Hermes** is the delivery layer: cron agents, Telegram messages, daily accountability
- **Claude Code** is the execution layer: code, workflows, analysis, project management
- **The Vault** is the shared memory layer: both agents read and write here
- **GitHub** (`https://github.com/manyhatsautomations/Manyhat`) is the backup and version history
